Performance Animation Demo

Phase: Composited Animation

This demo shows the difference between composited and non-composited animations:

  • Composited (First 5 seconds): Uses `transform` and `opacity` - GPU accelerated, smooth performance
  • Non-Composited (After 5 seconds): Uses `left`, `width`, `height` - Main thread, may cause jank
Performance Tip: Composited animations (transform, opacity) are handled by the GPU and don't block the main thread. Non-composited animations (layout properties) require reflow/repaint and can cause performance issues.
